什么是Functional Programming?

​ Functional Programming是一种编程泛型(programming paradigm), 就像Object-oriented Programming(OOP)一样,就是一种写程序的方法,这些方法告诉我们如何思考及解决问题.

Functional Programming基本条件

​ 跟OOP一样不是所有的语言都支持FP,能够支持FP的语言至少需要符合函数为一等公民的特性.

函数为一等公民(First Class)

​ 一等公民就是指跟其他数据类型具有同等地位, 也就是说函数能够被赋值给变量, 也能当做参数传递, 也可以当做一个函数的返回值.

​ 函数能够被赋值给变量

​ 函数能够被当做参数传入

​ 函数被当做另一个函数的返回值

Functional Programming重要特性

Expression, no Statement

​ Functional Programming都是表达式(Expression), 不会是陈述式(Statement).

​ 表达式是一个运算过程,一定会有返回值,例如执行一个function

​ 陈述式是表现某个行为,例如赋值给一个变量

​ Functional Programming最早是为了做运算处理不管I/O, 而Statement通常都是属于对系统I/O的操作,所以Functional Programming很自然的不是Statement.

在实际事务中不可能完全没有I/O操作, Functional Programming 只要求对I/O操作限制到最小,不要有不必要的I/O行为,尽量保持运算过程的单纯.

狭义的I/O是值 内存与设备之间的数据交换；广义的I/O值任何的数据交换都叫I/O

Pure Function

​ Pure Function是指一个Function给予相同的参数,永远会返回相同的返回值, 并且没有明显的副作用(Side Effect)

​ 不论执行slice多少次,返回值都是相同的,并且除了返回一个值外并没做任何事,所以slice是一个Pure Function

​ 每次执行splice都会影响arr的值,导致每次的结果都不同,这就不是一个Pure Function

Side Effect

​ Side Effect是指一个Function做了和本身运算返回值没有关系的事.

​ 例如:

  • 发送Http Request
  • 在控制台打印内容或log
  • 获取使用者的输入
  • Dom操作
  • …..

​ Functional Programming强调没有Side Effect, 也就是说Function要保持纯粹, 只做运算并返回一个值, 没有其他的额外行为.

Referential transparency

​ Pure Function不管外部环境如何, 只要参数相同,函数执行的返回结果必定相同. 这种不依赖任何外部状态, 只依赖传入参数的特性被称为 引用透明性(Referential transparency)

利用参数保存状态

​ findIndex用来查找数组中元素的位置, 在findIndex 中多设置了一个参数用来保存当前找到第几个Index的状态, 这就是利用参数保存状态

可维护性高

​ 由于pure function等特性,执行结果不依赖外部状态,且不会对外部环境有任何操作,使Functional Programming能更好的去除错误及编写单元测试.

易于并行/平行处理

​ Functional Programming易于做并行/平行处理(Concurrency/Parallel)处理, 因为基本上只做运算不碰I/O, 再加上没有Side Effect的特性, 所以不需要担心deadlock等问题
